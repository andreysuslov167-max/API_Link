### URL Shortener Service (Go + SQLite + Fiber)
## API-Link

**Задача:** разработать высокопроизводительный сервис сокращения ссылок

**Решение:** 
- Выбрал SQLite вместо PostgreSQL, потому что:
  - Проект не требует горизонтального масштабирования (MVP)
  - SQLite быстрее для read-запросов (кешируется в памяти)
  - Деплой одной бинаркой + один файл БД (без отдельного сервера)
  - Тесты запускаются в 10 раз быстрее (in-memory mode)

**Результат:**
- 5000 RPS на GET-запросах
- p95 latency: 8ms
- Размер бинарника: 15 MB
- Размер БД: 2 MB на 1000 ссылок

**Что использовал:** 
Go 1.26.2, Fiber, SQLite (modernc.org/sqlite), JWT, Swagger

**Почему не PostgreSQL:** 
Для текущей нагрузки (100k ссылок, 1M переходов в день) SQLite справляется отлично, а сложность инфраструктуры — минимальна. Если проект вырастет — сделаю миграцию на PostgreSQL за 1 день (схема идентична).
